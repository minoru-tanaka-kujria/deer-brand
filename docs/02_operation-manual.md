## 1. 共通ログインモーダル

### 画面の目的
全ページ共通で表示されるログイン／新規登録モーダル。`js/auth-modal.js` がページの `</body>` 直前に動的注入する。

### 対応認証プロバイダー
| プロバイダー | 方式 | 実装状態 |
|---|---|---|
| Google | Firebase signInWithPopup + GoogleAuthProvider | 実装済み |
| Apple | Firebase signInWithPopup + OAuthProvider("apple.com") | 実装済み |
| LINE | OAuth 2.0リダイレクト（`/line-callback` へ遷移） | 実装済み（コールバック処理は別途） |
| メール/パスワード | Firebase Email/Password | 実装済み |

### 操作手順（正常系）

#### Google ログイン
1. `DeerAuth.openModal()` またはログインボタンをクリックしてモーダルを開く
2. 「Googleでログイン」ボタンをクリック
3. ブラウザのGoogle認証ポップアップが開く
4. Googleアカウントを選択してログインを承認
5. モーダルが自動で閉じ、ナビゲーションバーがアカウント表示に切り替わる
6. `window.onDeerLogin(user)` コールバックが実行される（ページ固有の処理）

#### メール新規登録
1. モーダルを開く
2. メールアドレス欄に入力（例: `test@example.com`）
3. パスワード欄に6文字以上を入力（例: `pass1234`）
4. 「新規登録」ボタンをクリック
5. Firebaseでユーザー作成
6. `_giveWelcomeCoupon(uid)` が呼ばれる（**注意: 現在 console.log のみ、実際の付与処理なし**）
7. モーダルが閉じる

#### メール ログイン
1. モーダルを開く
2. 登録済みメールアドレスとパスワードを入力
3. 「ログイン」ボタンをクリック

### 異常系・エラーケース

| 操作 | エラー内容 | 表示メッセージ | 表示場所 |
|---|---|---|---|
| メールログイン | 未登録アドレス | このメールアドレスは登録されていません | `#authError` div |
| メールログイン | パスワード誤り | パスワードが正しくありません | `#authError` div |
| メールログイン | 形式不正メール | メールアドレスの形式が正しくありません | `#authError` div |
| メールログイン | 連続失敗 | しばらく後に再試行してください | `#authError` div |
| 新規登録 | 既存アドレス | このメールアドレスはすでに登録済みです | `#authError` div |
| 新規登録 | パスワード5文字以下 | パスワードは6文字以上にしてください | `#authError` div（クライアントバリデーション） |
| Google/Apple | Firebase未設定 | Firebase設定が必要です（alert） | ブラウザalert |
| Google/Apple | ポップアップブロック | Googleログインに失敗しました: (errorメッセージ) | `#authError` div |

### テストデータ例

| 区分 | メールアドレス | パスワード | 期待結果 |
|---|---|---|---|
| 正常値 | `newuser@deer-test.com` | `test1234` | 登録成功 |
| 正常値（境界） | `a@b.cc` | `123456`（6文字） | 登録成功 |
| 不正値 | `notanemail` | `pass1234` | 形式エラー |
| 不正値 | `test@example.com` | `12345`（5文字） | パスワード短すぎるエラー |
| 不正値 | 空欄 | 空欄 | 「入力してください」エラー |

## 3. 注文フォーム 9ステップ詳細

### 画面の目的
ペット写真をAIアートに変換してグッズを注文する。進捗バー付きの多段フォーム構成。

### ステップ全体像

| ステップ | セクションID | 内容 |
|---|---|---|
| Step 1a | `step-lookbook` | ルックブックから商品を選ぶ |
| Step 1b | `step-item-select` | アートスタイルから商品を選ぶ（art-stylesページ経由） |
| Step 2 | `step-placement` | プリント配置を選ぶ |
| Step 3 | `step-color` | カラーを選ぶ |

| Step 9 | `step-confirm` | 注文内容確認・割引適用・注文確定 |

### Step 1a: ルックブックから選ぶ（`step-lookbook`）

**目的:** モデル着用イメージから商品・スタイルを選択する

**操作手順（正常系）:**
1. ページを開くと3カラムのルックブックグリッドが表示される（JSで動的生成）
2. 気に入った商品カード（`lookbook-card`）をクリックして選択
3. 選択済みカードにテラコッタ色のボーダーと右上のチェックバッジが表示される
4. 「この商品を選ぶ」ボタン（`#btnLookbookNext`）が活性化する
5. ボタンをクリックして Step 2 へ進む

**入力なし / 選択のみ**

**異常系:**
| 状況 | 挙動 |
|---|---|
| 何も選択せず次へボタン | ボタンが `disabled` 状態のため押下不可 |
| 全スタイルを見る | `art-styles` ページへ遷移（Step 1b に入る） |

### Step 2: プリント配置を選ぶ（`step-placement`）

**目的:** アートの印刷位置を選択する。配置によって価格が変わる

**操作手順（正常系）:**
1. プレースメントカードグリッド（`#placementGrid`）が表示される（JSで動的生成）
2. 配置オプションをクリックして選択（テラコッタボーダーで選択状態表示）
3. 各カードには配置名・説明・追加料金が表示される
4. 「次へ — カラー選択」ボタン（`#btnPlacementNext`）が活性化
5. 「戻る」クリックで前ステップへ戻る

---

### Step 3: カラーを選ぶ（`step-color`）

**目的:** 商品のカラーを丸型スウォッチから選択する

**操作手順（正常系）:**
1. 商品に対応したカラースウォッチ（`color-swatch`）が表示される（JSで動的生成）
2. スウォッチをクリックして選択
3. 選択済みスウォッチにテラコッタ色のボーダーが表示される
4. 「次へ — サイズ選択」ボタン（`#btnColorNext`）が活性化

### Step 4: サイズを選ぶ（`step-size`）

**目的:** アパレル商品はS〜XXLのサイズ選択、スマホケースはブランド・機種選択

**操作手順（正常系 / アパレル）:**
1. `#sizeSelectWrap` が表示され、サイズボタン（`size-btn`）が並ぶ
2. 希望サイズをクリックして選択
3. 「サイズガイドを見る」をクリックすると着丈・身幅・肩幅の表が展開する
4. 「次へ — 写真アップロード」ボタン（`#btnSizeNext`）が活性化

**操作手順（正常系 / スマホケース）:**
1. `#modelSelectWrap` が表示される
2. まずブランド（`#modelBrandOptions`）を選択
3. 続いて機種（`#modelOptions`）を選択
4. 選択した機種が `#modelSelected` に確認表示される

**テストデータ例:**
| 区分 | 入力値 | 期待結果 |
|---|---|---|
| 正常値（アパレル） | M | 選択完了、次へ進める |
| 正常値（ケース） | iPhone → iPhone 15 Pro | 選択完了、次へ進める |
| 異常系 | 未選択のまま | `#btnSizeNext` が disabled |

### Step 5: 写真アップロード（`step-photo`）

**目的:** ペットの写真を頭数分アップロードし、名前入れオプションを設定する

**操作手順（正常系）:**

#### 1. 頭数選択
1. `#petCountGrid` から頭数カード（1〜4頭）をクリック
2. 追加料金の目安が各カードに表示される:
   - 1頭: 追加料金なし
   - 2頭: +¥1,000
   - 3頭: +¥1,500
   - 4頭: +¥2,000
3. 選択した頭数に応じて `#uploadZonesContainer` に写真アップロードエリアが生成される

#### 2. 写真アップロード
1. `input#fileInput`（`accept="image/*"`）からファイル選択ダイアログを開く
2. ペット写真ファイルを選択
3. アップロードゾーン内にプレビューが表示される（`.preview-img`）
4. 頭数分すべてのゾーンに写真がアップロードされると次へ進める

**推奨写真条件（画面上のヒントより）:**
- 顔がはっきり写っている正面〜やや斜めの写真
- 明るい場所で撮影された高解像度の写真
- 背景がシンプルなもの
- 1枚に1頭のみ

#### 3. 名前入れ（任意）
1. 「名前を入れる」トグルスイッチ（`#nameToggle`）をONにする
2. `#nameInputSection` が展開する
3. 頭数分のペット名入力フィールド（`#petNameFields`）が表示される
4. 各ペット名を入力（例: `モモ`、`ハナ`）
5. フォント選択（5種類）:
   - ゴシック（Noto Sans JP）
   - 明朝（Noto Serif JP）
   - 丸ゴシック（Zen Maru Gothic）
   - 手書き風（Yomogi）
   - 筆記体（Dancing Script）
6. 位置調整: `#positionPreview` エリアの矢印ボタン（上下左右）で名前の配置位置を微調整

4. 「アートを生成する」ボタン（`#btnPhotoNext`）をクリック

**異常系・エラーケース:**
| 状況 | 挙動 |
|---|---|
| 写真未アップロードで次へ | `#btnPhotoNext` が disabled |
| 名前トグルONで名前未入力 | バリデーション動作（詳細はソースコードJSによる） |
| ファイルタイプ | `accept="image/*"` のみ（ファイルサイズ・形式の詳細バリデーションなし→ギャップ分析参照） |

**テストデータ例:**
| 区分 | データ | 期待結果 |
|---|---|---|
| 正常値 | JPEG 1ファイル、1頭選択 | プレビュー表示、次へ進める |
| 正常値（複数） | PNG 2ファイル、2頭選択 | 両方プレビュー表示、次へ |
| 境界値 | 4頭選択、4ファイル | 全アップロード完了で次へ |
| 不正値 | PDF等の非画像ファイル | OSのファイルピッカーで除外（accept属性） |

### Step 8: お支払い方法（`step-payment`）

**目的:** 決済手段を選択し、カード情報を入力する

**選択肢:**
| data-method | 表示名 | 備考 |
|---|---|---|
| `card` | クレジットカード | デフォルト選択 / Stripe Elements使用 |
| `applepay` | Apple Pay | Stripe Automatic Payment Methods対応 |
| `googlepay` | Google Pay | Stripe Automatic Payment Methods対応 |

**操作手順（正常系 / クレジットカード）:**
1. 「クレジットカード」が選択された状態（デフォルト）
2. `#card-element` にStripe Elementsのカード入力フォームが表示される
3. カード番号・有効期限・CVC・郵便番号を入力
4. ログイン済みの場合、`#saveCardLabel` が表示され「このカードを保存する」チェックボックスが使える
5. ログイン済みで保存済みカードがある場合、`#savedCardsList` に既存カードが表示される
6. 「注文内容を確認する」ボタン（`#btnPaymentNext`）をクリック

**Stripeテストカード:**
| カード番号 | 用途 |
|---|---|
| `4242 4242 4242 4242` | 通常成功 |
| `4000 0000 0000 9995` | カード残高不足エラー |
| `4000 0025 0000 3155` | 3Dセキュア認証が必要 |

**有効期限:** 将来の任意の日付（例: `12/28`）  
**CVC:** 任意3桁（例: `123`）

**異常系:**
| 状況 | 挙動 |
|---|---|
| カード番号不完全 | Stripe側でリアルタイムバリデーション、`#card-errors` にエラー表示 |
| 期限切れカード | Stripe側でエラー表示 |

### Step 9: 注文内容確認（`step-confirm`）

**目的:** 注文内容を最終確認し、割引を適用して注文を確定する

**表示内容（`#confirmDetails`）:**
- 選択した商品・スタイル・配置・カラー・サイズ
- ペット頭数・名前
- お届け先住所
- 小計・割引額・合計（`#confirmTotal`）

**Instagram フォロー割引（`#igDiscountBox`）:**
1. `@deer_brand_official` をフォロー
2. 「フォロー中」状態のスクリーンショットをアップロード（`#igScreenshotInput`、`accept="image/*"`）
3. プレビュー画像が `#igPreviewImg` に表示される
4. `#igVerifySection` の「スクリーンショットを確認して割引を適用」ボタンが表示される
5. クリックで `/api/verify-ig-follow` へPOST（base64画像データを送信）
6. 検証成功（`verified: true`）なら ¥500 OFF が適用される
7. `#igStatus` に結果メッセージを表示

**クーポンコード（`#couponBox`）:**
1. `#couponCode` 入力欄にコードを入力（大文字変換される）
2. 「適用」ボタン（`#couponApplyBtn`）をクリック
3. `/api/validate-coupon` へ POST（`{ code, subtotal }`）
4. 検証成功なら割引が適用され `#couponStatus` に確認メッセージ表示

**テストクーポンコード:**
| コード | 割引 | 最低購入額 | 備考 |
|---|---|---|---|
| `DEER2026` | ¥1,000 固定 | ¥3,000 | 記念クーポン |
| `WELCOME500` | ¥500 固定 | ¥0 | 新規登録特典 |
| `SUMMER10` | 10% | ¥5,000 | サマーセール |
| `GIFT500` | ¥500 固定 | ¥2,000 | ギフト購入 |

**注文確定フロー:**
1. 「注文を確定する」ボタン（`#btnOrder`）をクリック
2. `placeOrder()` 関数が実行される
3. `/api/stripe-payment-intent` へ POST して PaymentIntent を作成
4. Stripe.js で決済を実行
5. 成功後、`/api/create-order` へ POST して注文をFirestoreに保存
6. `status: "pending"` で作成、Stripe Webhook受信後に `status: "preparing"` に更新
7. 完了画面（`complete-screen`）に注文ID（`DEER-YYYYMMDD-XXXX`形式）が表示される

**異常系:**
| 状況 | 挙動 |
|---|---|
| 決済失敗（残高不足等） | Stripe側エラー、`#card-errors` または別途エラー表示 |
| ネットワークエラー | エラーメッセージ（実装によってUIが異なる場合あり） |
| IG検証失敗 | `#igStatus` に失敗理由を表示、割引未適用 |
| クーポン無効 | `#couponStatus` に理由を表示 |

## 4. マイページ（account.html）

### 画面の目的
ログインユーザーの情報・注文履歴・クーポン・住所・カードを一元管理する

### 認証チェック
- `onAuthStateChanged` で監視
- 未ログイン時: 「ログインが必要です」と登録・ログインボタンを表示
- ログイン済み: Firestoreから `users/{uid}` のデータを取得して画面を構築

### 画面構成（セクションカード）

| セクション | 内容 |
|---|---|
| アカウント情報 | 表示名・メール・プロバイダーアイコン・プロフィール保存ボタン |
| お届け先住所 | `savedAddresses` 配列を表示。注文時に自動保存される |
| お支払い方法 | Stripe保存カード表示（`#savedCards`）・カード追加ボタン |
| 保有クーポン | `availableCoupons` 配列を表示 |
| 注文履歴 | `orders` 配列（最大5件）を表示 |

### 操作手順（正常系）

#### プロフィール保存
1. 「表示名」フィールド（`#editDisplayName`）を編集
2. 「プロフィールを保存」ボタンをクリック
3. `window.saveProfile(uid)` が実行される
4. Firestoreの `users/{uid}.displayName` を `updateDoc` で更新
5. 成功時: 「プロフィールを保存しました」のトーストが表示される（黒背景、2.5秒後に消える）
6. 失敗時: 「保存に失敗しました」のトースト（赤背景）

#### ログアウト
1. 「ログアウト」ボタン（`.logout-btn`）をクリック
2. `signOut(auth)` が実行される
3. `index.html` へリダイレクト

#### カード追加（現在の実装）
1. 「カードを追加」ボタン（`.add-card-btn`）をクリック
2. `openStripeCardSetup()` が実行される
3. **現在は alert 表示のみ** — 「カード追加機能は現在準備中です。注文時にカードを登録するとここに保存されます。」

### 注文ステータス表示

| status値 | 表示テキスト | スタイル |
|---|---|---|
| `preparing` | 制作中 | 黄色背景 (`#fff3cd`) |
| `shipped` | 発送済み | 水色背景 (`#d1ecf1`) |
| `delivered` | 配達完了 | 緑背景 (`#d4edda`) |
| その他 | — | 表示なし（対応なし） |

### 異常系・エラーケース

| 状況 | 挙動 |
|---|---|
| 未ログインでアクセス | 「ログインが必要です」画面を表示 |
| Firestore取得エラー | `null` として扱い、各フィールドが空の状態で表示 |
| プロフィール保存失敗 | 赤トーストで「保存に失敗しました」 |
| 注文履歴なし | 「まだ注文がありません。はじめての注文をする →」を表示 |
| 住所なし | 「保存された住所がありません。注文時に住所を入力すると自動保存されます。」 |

### テストデータ例

| 項目 | テストデータ |
|---|---|
| 表示名 | `山田 花子`（変更して保存テスト） |
| 保有クーポン | Firestoreに `availableCoupons: [{code: "DEER2026", ...}]` を設定 |
| 注文履歴 | Firestoreに `orders: ["DEER-20260401-ABCD"]` を設定 |
